# Project Summary
The project is a web application named Social Auto, designed to facilitate content generation and campaign planning through AI technologies. It provides users with tools such as a landing page, post and flyer generators, and a campaign planner, all while ensuring complete user authentication.

# Project Module Description
- **Backend**: Handles API requests, authentication, and data management.
- **Frontend**: User interface components, routing, and state management.
- **Services**: Contains business logic for various functionalities including user management and content generation.
- **Schemas**: Defines data structures used throughout the application.
- **Middlewares**: Manages request processing and validation.

# File Description Inventory
- **README.md**: Provides an overview and instructions for the respective module.
- **alembic.ini**: Configuration file for database migrations.
- **main.py**: The main entry point for the backend application.
- **package.json**: Manages frontend dependencies and scripts.
- **vite.config.ts**: Configuration for the frontend build tool.

# Technology Stack
- **Backend**: Python, FastAPI, SQLAlchemy, Alembic
- **Frontend**: React, TypeScript, Vite, Tailwind CSS
- **Database**: PostgreSQL

# Usage
To set up the project:
1. Install dependencies:
   - For backend: run `pip install -r requirements.txt`
   - For frontend: run `npm install` or `pnpm install`
2. Build the frontend:
   - Run the build command specified in the package.json.
3. Start the backend server:
   - Execute the main entry point script.
